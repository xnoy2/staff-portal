📋 System Rules Summary
Rule Detail Who Controls
Payroll period Based on configurable cut-off day (default: 25th). Period = (cutoff+1) of prev month → cutoff of current month. Admin Manager HR
Included entries Only time entries with status = approved and clock_in within the period are included. Pending or rejected entries are excluded. System
Hour split — Standard shift ≤8h → all regular. >8h → first 8h regular, remainder = OT (is_overtime flag set automatically). System
Hour split — OT/RDOT shift If ot_type is set (approved OT request), 100% of hours count as overtime regardless of duration. System
Pay calculation Gross = (Regular hrs × Rate) + (OT hrs × Rate × 1.5). Net = Gross − Deductions. System
Duplicate prevention If a payslip for the same user + period already exists, it is skipped during generation. Shown in success message. System
Deductions Can only be added/edited on draft payslips. Removed once approved. Manager HR
Approval lock Approved payslips cannot be edited or deleted via the portal. Any correction requires Admin intervention at database level. Admin only
Staff excluded from payroll Admin-role users are never included in payroll runs. Inactive staff (is_active = false) are also excluded. System
Staff notification Staff receive an in-app notification when their payslip is approved. Notification includes period dates and gross pay. System
CSV export scope Only approved payslips are exported. Draft payslips are excluded from the CSV to ensure accuracy. Admin Manager HR

📌 Comparison with Sample Retail Payroll Flow

The sample flowchart uses a traditional batch payroll processor / vendor model (HR enters data → Payroll vendor produces payments). Our portal is a fully self-contained digital system — there is no external payroll vendor. Key differences: (1) clock-in/out is digital and real-time, not manually entered; (2) hour calculations are automatic; (3) payroll approval is done inside the portal by Manager/HR, not sent to an external processor; (4) the portal does not yet handle automated bank payments — the CSV export is used for that step externally.
